## What was built

An interactive 3D visualization of the inner solar system and its near-Earth objects, driven entirely by the `data/` dataset.

**Architecture**
- **`scripts/preprocess.mjs`** (build-time) reads the read-only `data/` JSON and packs orbital elements into compact `Float32` binary blobs (~1.9 MB vs 16 MB source) plus trimmed JSON for the UI, written to `public/data/` → copied verbatim into `dist/data/`.
- **GPU Kepler propagation** (`src/bodies.ts`): a vertex shader solves Kepler's equation (elliptic / hyperbolic / parabolic) for all ~46k bodies every frame from one time uniform — this is what makes 42k asteroids + 4k comets animate smoothly.
- Propagation uses each body's time of perihelion passage `tp` (`M = n·(t−tp)`), verified to reproduce the dataset's own mean anomaly at epoch.

**Base requirements** ✅ Sun + 8 planets (with orbit rings, ranked-correct sizes) + full ~42k NEOs + ~4k comets, all computed from elements · interactive time (play/pause, ±speeds, 1950–2125 scrub, jump-to-date, Today) · real data loaded via `fetch` at runtime · orbit-camera, starfield, legible HUD.

**Open-ended directions — all implemented:** filter & search (name, PHA, Sentry, class, diameter, MOID) · single-object detail view with physical/orbital data and Earth close-approach history · PHA/Sentry highlighting · color-by hazard/inclination/eccentricity + size encoding · comets overlay (open orbits handled correctly) · CNEOS Sentry impact-risk join · focus & follow · shareable deep links.

**Hard rules** ✅ `data/` untouched · no external network calls (three bundled, system fonts, inline-SVG favicon) · `pnpm install && pnpm build` succeeds cleanly into a self-contained `dist/` · all relative URLs · everything stays in-directory.
